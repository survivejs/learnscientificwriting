# Design improvement plan

Implement the visual improvements in focused commits so each change can be reviewed independently.

1. Refine the reading surface and article typography.
   - Make book pages feel more editorial and less default.
   - Improve prose rhythm, chapter heading scale, and page background.

2. Polish the side table of contents.
   - Add a clear "On this page" label.
   - Improve active state, indentation, and hover/readability.

3. Turn previous/next navigation into deliberate cards.
   - Make chapter progression easier to scan.
   - Improve mobile affordance without covering content awkwardly.

4. Upgrade the homepage hierarchy.
   - Clarify the offer and add a direct reading CTA.
   - Make the table of contents section feel more intentional.

5. Fix small visual rough edges.
   - Correct the highlight stylesheet fallback typo.
   - Re-check build output and validation after the full pass.
